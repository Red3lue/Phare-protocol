Phare

A citizen-bonded sighting registry for sanctioned tankers.

ETHPrague 2026 · solar-punk

The problem

  • The shadow fleet moves sanctioned oil while evading detection.
  • They go AIS-dark, fake positions, re-flag every few months, transfer cargo at sea.
  • They cause sanctions failure, war financing, and oil spills with no insurer to sue.
  • Outside expensive satellite imagery and government surveillance, almost nobody can see them.

Solar-punk civic infrastructure, ecological accountability, skin-in-the-game over permission lists.

The idea

Anyone with a phone can:

  1. Photograph a suspicious tanker.
  2. Stake a small bond ($5 USDC) on the report being real.
  3. Submit it to a public on-chain registry.

Autonomous verifiers — and any human with the same bond — race to dispute fakes. Reports that survive a short challenge window become public, queryable records. Sanctioned vessels get their own permanent ENS identity.

Three actors

Reporter

Photographs vessels, submits sightings.

Phone, $5 USDC bond, an EOA.

Verifier

Watches the report stream, races to dispute fakes.

OpenClaw runtime, counter-bond, ENS identity.

Consumer

Reads the registry — journalists, NGOs, insurers.

Just a browser.

End-to-end flow

  1. PWA captures GPS · queries suggester for AIS vessels nearby.
  2. Empty list = the AIS-dark signature. Reporter enters IMO manually.
  3. Photo + metadata uploaded to Swarm.
  4. ReportRegistry.submit() on Base Sepolia: $5 USDC bond, opens a UMA assertion.
  5. Verifiers poll · fetch metadata · dispute or skip.
  6. Liveness expires → bond returned + small reward · or slashed if disputer wins.
  7. Minter mints imo-N.vessel.phare.eth with full sighting record.
  8. Orbital service requests satellite imagery → KMS-signed attestation lands on-chain.

spaceTEE inference (mocked)

Given a vessel image and the verifier logs that touched the same IMO, the TEE returns a destination prediction. Real spaceTEE is on the SpaceComputer roadmap — we ship the call site and a stub predictor.

Inputs

  • Image hash (Swarm bzz://)
  • Verifier-log refs (per-vessel)
  • Last-seen GPS + extrapolated heading

Output (signed JSON pinned to Swarm)

  • destination — port name string
  • confidence — 0.0 to 1.0
  • route_candidates — ranked alternatives
  • supporting_logs — bzz:// refs back to the verifier evidence
  • "mocked": true

Stub predictor: combine last-seen coordinate + extrapolated heading + a static map of likely destination ports for sanctioned-tanker routes. Crude on purpose. The contribution is the call site and the data contract — when spaceTEE ships, the plugin call site doesn't change.

Surfaced in ENS as vessel.orbital.tee.destination + vessel.orbital.tee.confidence; full JSON via vessel.orbital.tee.prediction.
