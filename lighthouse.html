Why wrap, why fuses

The legacy ENS Registry has a single mutable owner slot — the parent owner can clobber any subname at any time. That's not "ownership", that's permission.

NameWrapper turns each name into an ERC-1155 with fuses: burn-only permission bits. Once a fuse is burnt, it stays burnt. Forever.

CANNOT_UNWRAP

Burnt on the parent. Locks the name into wrapped form so we can burn fuses on its children.

Prerequisite for everything below.

PARENT_CANNOT_CONTROL

Burnt on each child. The parent — and Lighthouse itself — can no longer revoke or rewrite this child.

"Sovereign subname" guarantee.

CANNOT_TRANSFER

Burnt on vessel children only. Locks the subname token to Lighthouse forever — vessels are unsellable, unmovable.

Identity that cannot be hijacked.

Mirror-image trust models

The same code, two opposite outcomes — and the asymmetry is the point.

A vessel cannot hold a wallet → contract permanently owns the token. A verifier is a wallet → principal owns the token. Same fuse system, opposite arrangements.

One contract: Lighthouse.sol

contract Lighthouse {
    bytes32 public immutable vesselParent;    // namehash("vessel.phare.eth")
    bytes32 public immutable verifierParent;  // namehash("verifier.phare.eth")
    address public immutable reportRegistry;

    uint32 constant FUSES_VERIFIER = PARENT_CANNOT_CONTROL;
    uint32 constant FUSES_VESSEL   = PARENT_CANNOT_CONTROL
                                   | CANNOT_TRANSFER
                                   | CANNOT_UNWRAP;

    function nameVessel(uint256 imo, string calldata swarmRef)
        external onlyRegistry returns (bytes32);          // owner = self

    function recordSighting(uint256 imo, string calldata swarmRef,
                            uint32 sightings, uint32 disputed)
        external onlyRegistry;

    function recordOrbital(uint256 imo, string calldata image,
                           bytes32 imageHash, string calldata teePrediction)
        external onlyRegistry;

    function enrollVerifier(string calldata handle,
                            string calldata policyURI,
                            string calldata soulURI)
        external returns (bytes32);                        // owner = msg.sender
}

~80 lines including the imo-label formatter and helper internals. Operator-approved on both parents during one-shot setup. Parent owner key keeps the parent NFTs as an escape hatch.
